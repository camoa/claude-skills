# Design System Converter

Convert design systems to production code. Currently supports HTML pages with metadata comments as input and Drupal Radix sub-themes with SDC components as output.

## Installation

```bash
/plugin install design-system-converter@camoa-skills
```

## Usage

```bash
# Full guided wizard (recommended for first use)
/convert-to-radix

# Quick mode with minimal questions
/convert-to-radix-quick
```

## What It Does

Takes branded HTML pages (generated by `brand-content-design` or any HTML with metadata comments) and produces:

- Complete Radix 6.0.2 sub-theme with Bootstrap SCSS
- SDC components (`.component.yml`, `.twig`, `.scss`)
- Twig template overrides mapping entity fields to component props
- Layout Builder composition instructions
- Drupal Icon API icon pack from inline SVGs
- Config export YAML (block types, fields, views, LB styles)

## Conversion Workflow

| Phase | Action |
|-------|--------|
| Discovery | Find HTML pages and design system |
| Analysis | Parse metadata, classify patterns, extract tokens, inventory Drupal backend |
| Review | Present plan with Drupal opinions, resolve ambiguities |
| Generation | Create Radix sub-theme, SDC components, template overrides, LB config |
| Report | File manifest, module requirements, composition instructions |

## Architecture

- **Metadata-driven**: Parses `<!-- component: -->` / `<!-- prop: -->` / `<!-- slot: -->` comments dynamically
- **Shared analysis + target-specific generators**: Analyzer is target-agnostic (reusable by future Canvas/Next.js targets)
- **6px threshold framework**: Maps design tokens to Bootstrap SCSS variables
- **Drupal backend inventory**: Scans `config/sync/` for existing entities to maximize reuse
- **External guide ecosystem**: Domain knowledge at `https://camoa.github.io/dev-guides/`, discovered via `llms.txt`
